Workflow AI Terapan

Bukan cuma prompt tunggal — ini rangkaian langkah yang bisa langsung diadaptasi ke pekerjaan kamu.

coding

Otomasi code review di pull request

Menyisipkan pemeriksaan AI sebelum review manusia, agar reviewer fokus ke isu yang lebih penting.

  1. Trigger workflow saat pull request dibuka
  2. AI menganalisis diff untuk bug, security, dan style
  3. Hasil ditulis sebagai komentar terstruktur di PR
  4. Reviewer manusia memvalidasi & memutuskan merge
data

Laporan data mingguan otomatis

Dari query database ke narasi insight yang siap dikirim ke stakeholder.

  1. Jadwalkan query ekstraksi data mingguan
  2. AI meringkas tren, outlier, dan perbandingan periode sebelumnya
  3. Hasil dirender ke format laporan (email/PDF)
  4. Analyst meninjau sebelum dikirim ke tim terkait
database

Review & optimasi query lambat

Menemukan dan memperbaiki query yang membebani database secara proaktif.

  1. Ambil daftar query paling lambat dari slow query log
  2. AI menganalisis execution plan dan mengusulkan indeks/refactor
  3. Uji perubahan di lingkungan staging
  4. Terapkan ke produksi dengan monitoring performa
infra

Dokumentasi topologi jaringan otomatis

Menjaga dokumentasi tetap sinkron dengan kondisi perangkat sebenarnya.

  1. Tarik konfigurasi terbaru dari seluruh perangkat
  2. AI menyusun ringkasan topologi & perubahan sejak versi terakhir
  3. Dokumentasi & diagram diperbarui otomatis
  4. Tim meninjau perubahan signifikan sebelum publish
devops

Triage insiden lebih cepat

Memangkas waktu dari alert masuk sampai tim tahu harus mulai dari mana.

  1. Alert monitoring masuk ke channel incident
  2. AI meringkas log & metrik terkait dalam satu paragraf
  3. AI menyarankan kemungkinan penyebab & runbook terkait
  4. On-call engineer melanjutkan investigasi dari titik itu
security

Audit keamanan config otomatis

Menjaga konfigurasi server & jaringan tetap sesuai baseline keamanan tanpa audit manual tiap minggu.

  1. Tarik konfigurasi terbaru dari perangkat/server yang relevan
  2. AI membandingkan dengan baseline & menandai penyimpangan berisiko
  3. Temuan diprioritaskan berdasarkan tingkat risiko
  4. Tim keamanan memvalidasi dan menjadwalkan perbaikan
qa

Generate & jalankan test case otomatis

Mempercepat siklus QA tanpa menulis semua test case dari nol.

  1. AI membaca deskripsi fitur/PR dan menyusun draf test case
  2. QA engineer meninjau dan menyesuaikan test case
  3. Test dijalankan otomatis di pipeline CI
  4. Hasil gagal diringkas AI untuk mempercepat debugging
karier

Menyusun studi kasus dari project selesai

Mengubah pekerjaan teknis yang sudah selesai jadi materi portofolio.

  1. Kumpulkan catatan, commit log, dan hasil project
  2. AI menyusun draf studi kasus: masalah, pendekatan, hasil
  3. Tambahkan angka/metrik dampak yang terukur
  4. Edit nada bahasa agar sesuai gaya personal
